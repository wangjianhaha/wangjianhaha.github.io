最初想搞个博客是去年四月，知道github很久却一直没动手，于是去注册了一个账号，按照网上的教程一步步做，结果死活搞不定，放弃。年初重新尝试，终于搞定，记得也是花了几乎一天时间，火急火燎写了第一篇《开耕》，后来又添了一个JavaScript笔记，就又完全荒废了。笔记还是习惯用笔记软件和手写。当时以为这套流程已经摸透了，naive。清明换了电脑，最近又想倒腾倒腾博客，终于动手。结果花了一天还差点搞不定，为何，原因就是这Hexo 3.0跟以前的2.x有点区别。想想还是挺怀疑自己的智商的。想想，写个新的教程吧，顺便理下流程。

第一步：博客环境

1、安装Node.js

因为hexo是基于Node.js开发的静态博客框架，所以必须安装。安装完就不用管它了。

2、安装Git

Git主要是用来执行git和hexo命令的，大部分操作都通过它进行。
打开Git的方式有很多，我习惯右键打开Git Bash Here 。

第二步：账号设置

1、注册GitHub账号

这个博客主要用的GitHub的GitHub Pages服务，搭配hexo框架实现博客功能。

2、建立仓库

所谓仓库，就是存放代码的地方，博客的所有代码以后都会存放在这里。
操作步骤是：用户首页：Repositories -> 右上角：New -> 输入Repository name
这个Repository name只能填username.github.io，比如我的username是wangjianhaha，那么对应的博客地址就是wangjianhaha.github.io，GitHub Pages服务也是通过这个地址实现的。

2、出现的问题

年初的时候这两条命令一点问题没有，可是现在，问题有两个：
1.执行hexo g会提示ERROR Try running: 'npm install hexo --save'，执行这条命令就没事了。
2.执行hexo d是没有效果的，既不配置文件也不提示错误，一点反应也没有，GitHub上也没有文件，一直为空，博客自然也打不开。
这一整天，就花在为什么执行hexo d没效果这个问题上了。
于是重头开始做，删掉本地所有文件，删掉SSH Key，删掉Repository，还是不行，重启再来。
冷静下来找原因，第一步第二步都没问题，那么问题肯定是在第三步上。第三步哪里有问题呢，是本地没配置好还是本地配置好了但是没上传。于是开始尝试。

提示running at http://0.0.0.0:4000/，浏览器打开这个地址竟然无法访问，本地没配置好？尝试输入http://localhost:4000/，以前就是访问这个地址，可以访问，是landspace主题，这时开始察觉到可能是Hexo版本的问题，网上的确有看到Hexo 3.0不兼容一些主题的文章。既然本地没问题，那么问题就在本地没上传，hexo d命令无效。
又重新操作了一遍，看到命令行里有hexo@3.0.1的信息。看来问题出在hexo d这条命令上。
终于在网上找到一篇Hexo 3.0遇到的问题，找到hexo d无响应的问题，需要在全局配置文件_config.yml里配置deploy值。配置如下：
